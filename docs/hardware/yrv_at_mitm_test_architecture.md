# YRV A/T MITM Test Architecture

Тестовая архитектура для стендовой и ограниченной дорожной проверки модуля-посредника между `ECM` и `A/T ECU` на Daihatsu YRV.

Основа по мануалам:

- В схеме двигателя `K3-VE / K3-VET` у `ECM` присутствуют линии `ATTX` и `ATRX`.
- В мануале `A/T` у `A/T ECU` присутствуют импульсные линии `SI`, `COMI` и `SPDO`.
- В мануале `A/T` есть код `82 / P1602`: линия связи с блоком управления двигателем.

Это делает пару `ATTX/ATRX <-> SI/COMI` главным кандидатом для перехвата, но перед врезкой в жгут это надо подтвердить прозвонкой и осциллографом.

## 1. Принцип

Модуль должен иметь три режима:

1. `Bypass`:
   - Штатная связь `ECM <-> A/T ECU` идет напрямую.
   - Это состояние по умолчанию при выключенном питании, сбросе ESP32, brownout и watchdog timeout.

2. `Monitor`:
   - Модуль только слушает обе линии через высокоомные приемники.
   - Разрыва штатной цепи нет.
   - Используется для снятия уровней, формы импульсов и направления до первого активного MITM.

3. `MITM`:
   - Линии разрываются аппаратно и заводятся в два независимых канала:
     - `ECM_TX -> MCU_RX`
     - `MCU_TX -> A/T_RX`
     - `A/T_TX -> MCU_RX`
     - `MCU_TX -> ECM_RX`
   - Алгоритм может прозрачно пересылать кадры и править только разрешенные поля.

## 2. Мой вариант электрической архитектуры

### 2.1 Питание

Не использовать голый `LM2596`-модуль как единственный автомобильный фронт.

Рекомендуемый тракт:

- `+12V IGN` от отдельного предохранителя `1 A`
- TVS на входе питания: automotive load-dump класса `SMBJ/SMCJ` по выбранной топологии и бюджету
- Защита от переполюсовки:
  - лучше ideal-diode на P-MOSFET
  - допустимо Schottky на тестовой ревизии, если устраивает падение
- LC-фильтр на входе:
  - `10-22 uH`
  - `47-100 uF low-ESR`
  - `100 nF` керамика рядом с DC/DC
- Automotive buck `12V -> 5V`
- LDO `5V -> 3.3V` для логики и `ESP32`

Разводка:

- силовую землю, землю реле и цифровую землю сводить звездой;
- аналоговые и чувствительные линии вести отдельно от катушки реле и DC/DC.

### 2.2 Коммутация fail-safe

Для тестовой ревизии:

- 2 независимых сигнальных реле `DPDT` либо 2 канала на релейной сборке с золотыми контактами
- состояние по умолчанию: `NC = штатный bypass`
- питание катушек подается только когда:
  - MCU прошел self-test;
  - внешняя логика watchdog разрешила активный режим;
  - уровень питания в норме

Важно:

- реле использовать только для входа в `MITM` и выхода из него на стоянке или при контролируемом тесте;
- не рассчитывать на бесшовное горячее переключение во время обмена.

### 2.3 Интерфейс линий связи

Я бы делал не на `TXB/TXS`, а на направленных буферах/transceiver.

Вариант A, предпочтительный:

- `SN74LVC2T45-Q1` x2
- один кристалл на направление `ECM -> MCU`
- второй на направление `MCU -> A/T ECU`
- если нужен полный двусторонний MITM по двум линиям, ставить отдельный переводчик на каждый направленный канал

Вариант B, проще и очень понятный:

- `5V -> 3.3V`: `SN74LVC1G17-Q1`
- `3.3V -> 5V`: `SN74AHCT125-Q1`

Плюсы варианта B:

- легко анализировать осциллографом;
- предсказуемое направление;
- меньше сюрпризов на шумной линии.

На каждой сигнальной ветви:

- последовательный резистор `33-100 Ohm` у драйвера
- ESD/TVS low-cap рядом с разъемом
- тест-пады до и после буфера

### 2.4 Прием и наблюдение без разрыва линии

Отдельно от MITM стоит предусмотреть режим `Monitor`.

Для него:

- пара высокоомных приемников `5V -> 3.3V`
- входы через `100k` + Schmitt buffer/компаратор
- этот тракт подключен всегда и не влияет на штатную связь

Это позволит сначала подтвердить:

- амплитуду;
- idle-level;
- скорость;
- направление;
- паузы между кадрами;
- допустимую дополнительную емкость линии.

### 2.5 Управление реле и fail-safe логика

Рекомендую не полагаться только на внутренний watchdog ESP32.

Узел управления:

- N-MOSFET low-side драйвер катушки
- flyback diode
- gate pulldown `10k`
- отдельный enable-сигнал `MITM_EN`
- внешний watchdog/supervisor, который снимает `MITM_EN`, если MCU завис или перестал стробировать

Поведение при аварии:

- watchdog timeout -> катушка обесточена -> `NC` bypass
- brownout `5V` или `3.3V` -> катушка обесточена -> `NC` bypass
- reset ESP32 -> катушка обесточена -> `NC` bypass

### 2.6 Разъемы и жгут

Для автомобильного теста лучше не паять модуль в штатный жгут намертво.

Лучший формат:

- промежуточный адаптер-врезка (`T-harness`)
- отдельные сервисные выводы:
  - `ECM_TX_RAW`
  - `ECM_RX_RAW`
  - `AT_TX_RAW`
  - `AT_RX_RAW`
  - `IGN`
  - `BATT`
  - `GND`
- сервисный разъем под логический анализатор

## 3. Текстовая схема

```text
IGN 12V
  |
 [FUSE]
  |
 [REVERSE POLARITY PROTECTION]
  |
 [AUTOMOTIVE TVS]
  |
 [LC FILTER]
  |
 [BUCK 5V] -----> [LDO 3.3V] -----> ESP32 / logic / watchdog
  |
  +-----> relay coils
  +-----> 5V line translators

ECM_ATTX ----o/ NC  RELAY_A  COM o---- A/T_COMI
             o/ NO             |
                               +--> 5V->3.3V receiver --> MCU_RX_A
MCU_TX_A --> 3.3V->5V driver --+

A/T_SI   ----o/ NC  RELAY_B  COM o---- ECM_ATRX
             o/ NO             |
                               +--> 5V->3.3V receiver --> MCU_RX_B
MCU_TX_B --> 3.3V->5V driver --+

Raw sniff taps:
ECM_ATTX --> high-Z monitor input
A/T_SI   --> high-Z monitor input
```

## 4. Компоненты по узлам

### Питание

- automotive buck `12V -> 5V`
- `5V -> 3.3V` LDO automotive/industrial
- входной TVS для бортсети
- предохранитель
- защита от переполюсовки

### Коммутация

- сигнальные реле с золотыми контактами, `NC` bypass

### Логические уровни

Предпочтительно:

- `SN74LVC2T45-Q1`

Или:

- `SN74LVC1G17-Q1`
- `SN74AHCT125-Q1`

### Защита сигнальных линий

- low-cap ESD array на каждую линию
- series resistor `33-100 Ohm`

### Надежность

- внешний watchdog/supervisor
- brownout monitor
- отдельный status LED для `BYPASS / MONITOR / MITM`

## 5. Что надо подтвердить по машине до активного MITM

1. Какая пара проводов реально соответствует `ATTX`, `ATRX`, `SI`, `COMI`.
2. Какой там idle level: `0/5V`, `open collector`, подтяжка, форма фронтов.
3. Какая скорость и протокол:
   - UART-подобный
   - нестандартная асинхронная линия
   - импульсный код без обычного UART framing
4. Допустима ли дополнительная задержка буфера в тракте.
5. Как ведет себя `A/T ECU`, если один кадр пропал, исказился или задержался.
6. Можно ли ограничиться не полным MITM, а подменой только одного направления.

## 6. Практический порядок тестирования

1. `Monitor only` на столе.
2. `Monitor only` в машине на холостом ходу.
3. Логирование на месте:
   - `P`
   - `R`
   - `N`
   - `D`
   - разные положения дросселя
4. Проверка реакции `A/T ECU` на пассивное присутствие модуля.
5. Прозрачный `MITM passthrough` без модификации.
6. Только потом ограниченная модификация одного параметра на стенде/закрытой площадке.

## 7. Ключевой вывод

Для первой ревизии лучше строить не "умный модуль коррекции", а "безопасный регистратор + прозрачный MITM с аппаратным bypass".

Если удастся стабильно пройти стадии:

- `Monitor`
- `Transparent passthrough`
- `Controlled single-field rewrite`

тогда уже можно двигаться к компактной ревизии платы.
