# ESP_Boost MITM Add-on Mapping

Адаптированная схема дооснащения для уже существующей платы `ESP_Boost`.

## Что уже занято на текущей плате

По схеме `ESP_Boost` и текущей прошивке заняты:

- `GPIO18` -> `rpmPin`
- `GPIO19` -> `vssPin`
- `GPIO22` -> `I2C SCL`
- `GPIO23` -> `I2C SDA`
- `GPIO25` -> `solPin`

Дополнительно по исходной текстовой схеме платы:

- `GPIO16 / RX2` и `GPIO17 / TX2` заведены на `L9637D` через LLC для `K-Line`

Вывод: узел перехвата `ECM <-> A/T ECU` лучше не встраивать в уже существующие цепи `16/17`, если вы хотите сохранить `K-Line`. Для MITM лучше выделить отдельные GPIO и отдельную дочернюю плату.

## Рабочая гипотеза по линиям YRV

- `ECM ATTX -> A/T ECU COMI`
- `A/T ECU SI -> ECM ATRX`

В мануале `A/T` OCR очень похож на `COM1`, но по смыслу и по контексту это выглядит как `COMI`.

## Рекомендуемая привязка к ESP32

Для дочернего MITM-узла взять отдельные GPIO:

- `GPIO26` -> `MITM_RX_FROM_ECM`
- `GPIO27` -> `MITM_TX_TO_AT`
- `GPIO32` -> `MITM_RX_FROM_AT`
- `GPIO33` -> `MITM_TX_TO_ECM`
- `GPIO4` -> `MITM_RELAY_EN`
- `GPIO5` -> `MITM_WD_STROBE`

Почему так:

- эти пины не заняты в текущей прошивке;
- `32/33` удобны как обычные цифровые входы/выходы;
- `4/5` можно оставить под управление и служебную логику;
- мы не трогаем уже разведенные узлы boost-контроллера.

## Топология дооснащения

Дочерняя плата должна врезаться между жгутом `ECM` и `A/T ECU`.

### Канал 1

- вход: `ECM ATTX`
- штатный выход: `A/T COMI`
- режим bypass: `ATTX` напрямую идет в `COMI`
- режим MITM:
  - `ATTX` идет в приемник `5V -> 3.3V`
  - после приемника идет в `GPIO26`
  - `GPIO27` через драйвер `3.3V -> 5V` формирует новый сигнал в `COMI`

### Канал 2

- вход: `A/T SI`
- штатный выход: `ECM ATRX`
- режим bypass: `SI` напрямую идет в `ATRX`
- режим MITM:
  - `SI` идет в приемник `5V -> 3.3V`
  - после приемника идет в `GPIO32`
  - `GPIO33` через драйвер `3.3V -> 5V` формирует новый сигнал в `ATRX`

## Текстовая схема

```text
                 EXISTING ESP_Boost BOARD
             +--------------------------------+
5V ----------+--------------------------------+----> daughterboard 5V
3V3 ---------+--------------------------------+----> daughterboard 3V3
GND ---------+--------------------------------+----> daughterboard GND
GPIO26 ------+--------------------------------+----> RX_FROM_ECM
GPIO27 ------+--------------------------------+----> TX_TO_AT
GPIO32 ------+--------------------------------+----> RX_FROM_AT
GPIO33 ------+--------------------------------+----> TX_TO_ECM
GPIO4 -------+--------------------------------+----> RELAY_EN
GPIO5 -------+--------------------------------+----> WD_STROBE
             +--------------------------------+


ECM ATTX ----o/ NC  RELAY_A  COM o---------------- A/T COMI
             o/ NO             |
                               +--> 5V->3.3V RX --> GPIO26
GPIO27 --> 3.3V->5V TX --------+

A/T SI   ----o/ NC  RELAY_B  COM o---------------- ECM ATRX
             o/ NO             |
                               +--> 5V->3.3V RX --> GPIO32
GPIO33 --> 3.3V->5V TX --------+
```

## Компоненты для дочернего узла

### Коммутация

- `2 x DPDT` сигнальных реле с золотыми контактами
- либо `1 x 4PDT`, если найдете адекватный корпус и ток катушки

### Согласование уровней

Предпочтительный простой вариант:

- `2 x SN74LVC1G17-Q1` для направлений `5V -> 3.3V`
- `1 x SN74AHCT125-Q1` для направлений `3.3V -> 5V`

Распределение:

- канал `ATTX -> GPIO26`: `LVC1G17-Q1`
- канал `SI -> GPIO32`: `LVC1G17-Q1`
- канал `GPIO27 -> COMI`: `AHCT125-Q1`, 1 буфер
- канал `GPIO33 -> ATRX`: `AHCT125-Q1`, 1 буфер

### Драйвер реле

- `AO3400A` или automotive-аналог, low-side
- flyback diode
- pull-down `10k` на gate

### Надежность

- внешний watchdog/supervisor
- `RELAY_EN` должен идти через аппаратное разрешение watchdog
- при любом сбое катушка без питания, состояние `NC`

### Защита сигналов

На каждую из 4 внешних линий:

- последовательный резистор `47-100 Ohm`
- low-cap TVS/ESD на массу рядом с входом

## Минимальный вариант без полной подмены

Если хотите первый этап с минимальным риском, можно сделать только `Monitor`:

- `ECM ATTX -> высокоомный приемник -> GPIO26`
- `A/T SI -> высокоомный приемник -> GPIO32`
- без реле
- без разрыва штатных линий

Это позволит подтвердить:

- что `COMI/SI` действительно парные линии к `ATTX/ATRX`
- уровни сигнала
- скорость и направление

## Чего я бы не делал на этой ревизии

- не использовал бы штатные `GPIO16/17`, если планируете сохранить `L9637D`
- не использовал бы `TXB0104 / TXS0104E`
- не заводил бы реле напрямую на GPIO
- не пытался бы переключать реле во время активного обмена

## Логика включения

1. Питание подано, реле выключены, `NC bypass`.
2. ESP32 грузится и начинает только слушать линии.
3. Внешний watchdog видит корректный строб.
4. Только после ручного разрешения включается `RELAY_EN`.
5. Узел переходит в прозрачный passthrough.
6. Лишь после этого можно тестировать модификацию кадров.

## Главная практическая мысль

Для вашей текущей платы `ESP_Boost` оптимально не переделывать основную схему, а добавить маленькую отдельную дочернюю плату:

- питание, `3V3`, `5V`, `GND` брать с основной платы;
- новые MITM GPIO брать отдельными проводами;
- сам разрыв автомобильных линий и fail-safe делать только на дочернем узле.
